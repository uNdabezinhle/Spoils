# Spoils – MVP Modules & Features (First Version)

**Document Purpose:** Define the scope of the first public version (MVP) of the Spoil gift shop application.

**Goal of MVP:** Launch a delightful, functional gift shopping experience with strong personalisation and the foundation for emotional gifting features.

---

## 1. Core Principle for MVP

**Focus on the main user journey:**
> Discover → Personalise → Purchase → Receive beautiful gift

Everything else should support or enhance this core loop.

**Guiding Rule:** If a feature doesn’t directly help a user send a thoughtful gift easily and beautifully in v1, it should be deferred.

---

## 2. Modules & Features for MVP

### Module 1: User Account & Authentication

**Must Have:**
- Email + password registration & login
- Social login (Google / Apple) – Recommended
- Password reset via email
- Basic profile (name, phone, default delivery address)
- Logout

**Nice to Have (if time allows):**
- Profile photo
- Multiple saved addresses

---

### Module 2: Gift Browsing & Discovery

**Must Have:**
- Homepage with hero banner + featured categories
- Product catalog with categories (Flowers, Hampers, Personalised Gifts, Experiences, etc.)
- Product detail pages with:
  - High-quality images
  - Description
  - Price
  - Delivery information
- Search functionality (basic)
- Filtering by price, category, and occasion
- “Featured” and “Popular” sections

**Out of Scope for v1:**
- Advanced AI recommendations
- Complex faceted search

---

### Module 3: Gift Customisation & Personalisation

**Must Have:**
- Add text message to gift
- Upload photo (single photo upload)
- Choose ribbon colour / wrapping style (from predefined options)
- Live preview of personalised gift (where technically feasible)
- Pre-written message templates by occasion

**Nice to Have:**
- Multiple photos
- Custom text on ribbon (if production allows)

---

### Module 4: Cart & Checkout

**Must Have:**
- Add to cart
- View and edit cart
- Apply basic promo codes (optional)
- Delivery address selection
- Delivery date selection (standard vs express)
- Order summary with breakdown (item + delivery + packaging)
- Secure checkout with **Paystack**
- Order confirmation screen + email

**Out of Scope for v1:**
- Multiple payment methods beyond Paystack
- Complex tax calculations

---

### Module 5: Order Management

**Must Have:**
- Order history list
- Order detail view with status
- Basic order tracking (Ordered → Processing → Shipped → Delivered)
- Reorder from past orders
- Download invoice / receipt

**Nice to Have:**
- Push notifications for order status updates

---

### Module 6: Occasion Reminders (Light MVP Version)

**Must Have:**
- Ability to add recipients + occasions (Birthday, Anniversary, Other)
- Set reminder timing (e.g. 14 days before)
- View list of upcoming occasions (“My People”)
- Basic reminder notification (push + in-app)

**Out of Scope for v1:**
- Smart gift suggestions from reminders
- Auto-gifting / approval flow
- Calendar view
- Advanced personalisation based on notes

> **Note:** Keep this module lightweight in v1. The goal is to plant the seed for recurring engagement.

---

### Module 7: Content & Presentation

**Must Have:**
- Beautiful product photography and presentation
- Clear delivery information and timelines
- “How it works” / FAQ section
- About Spoil page
- Terms & Privacy Policy

---

### Module 8: Admin / Backend (Internal)

**Must Have:**
- Basic product management (add/edit products)
- Order management dashboard
- Basic user management
- Manual order status updates

**Nice to Have:**
- Analytics dashboard (orders, revenue, top products)

---

## 3. Summary: MVP Feature Priority

| Priority | Module                        | Must Have? | Notes |
|----------|-------------------------------|------------|-------|
| High     | User Authentication           | Yes        | Foundation |
| High     | Gift Browsing & Discovery     | Yes        | Core experience |
| High     | Gift Customisation            | Yes        | Key differentiator |
| High     | Cart & Checkout               | Yes        | Revenue generation |
| High     | Order Management              | Yes        | Trust & retention |
| Medium   | Occasion Reminders (Light)    | Yes        | Strategic differentiator |
| Medium   | Content & Presentation        | Yes        | Brand experience |
| Low      | Advanced Admin Analytics      | No         | Can be added post-launch |

---

## 4. Out of Scope for First Version

- Full subscription box models
- Corporate gifting portal
- Advanced AI recommendations
- Group gifting / split payments
- International delivery
- Complex inventory management
- Live chat support
- Loyalty / points system
- AR try-on or advanced visualisation

---

## 5. Recommended Development Phases

**Phase 1 – Core Shopping Experience** (Highest priority)
- Authentication
- Browsing + Product pages
- Customisation
- Cart + Checkout
- Order history

**Phase 2 – Emotional Layer**
- Light Occasion Reminders
- Improved personalisation
- Better notification system

**Phase 3 – Growth & Retention**
- Full reminder + suggestion engine
- Subscription features
- Corporate tools

---

## 6. Success Criteria for MVP Launch

- Users can successfully browse, personalise, and purchase a gift end-to-end
- Orders can be fulfilled and tracked
- Basic occasion reminders are functional
- The experience feels warm, modern, and premium (brand-aligned)
